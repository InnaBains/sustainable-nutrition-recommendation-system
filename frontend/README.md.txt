Frontend Application – Sustainable Nutrition Recommendation System
------------------------------------------------------------

Overview
------------------------------------------------------------
This folder contains the React-based frontend application developed for
the Sustainable Nutrition Recommendation System.

The frontend provides the presentation layer of the prototype and allows
users to interact with recommendation results generated by the backend
and database decision engine. The interface was implemented as a
single-page web application using React and Vite.

The frontend communicates with the Spring Boot backend through REST API
requests and presents recommendation outputs in an accessible dashboard
and meal-planning interface.


Technology Stack
------------------------------------------------------------
The frontend application was developed using:

- React
- JavaScript (ES6)
- Vite
- HTML5
- CSS

React was selected because its component-based structure supports
dynamic user interfaces, reusable elements and state-driven rendering.


Main Interface Structure
------------------------------------------------------------
The frontend is organised around three primary views accessed through
tab navigation.


1. Dashboard / Home
------------------------------------------------------------
The dashboard provides an overview of the system state, including:

- summary of the current weekly meal plan
- pantry expiry indicators
- sustainability and nutrition quick statistics
- contextual assistant tips based on pantry and recommendation data

This view gives the user an immediate summary of pantry urgency,
recommendation quality and environmental impact.


2. Weekly Meal Plan
------------------------------------------------------------
The weekly meal plan view presents recommended meals in a grid organised
by days of the week and meal categories (breakfast, lunch, dinner).

Features include:

- weekly plan generation
- nutrition and CO2e indicators for each recipe
- recipe details displayed in a modal window
- filter controls to refine the recommendation set

Displayed recipes are deduplicated, filtered, classified into meal types
and allocated across the weekly grid.


3. Pantry Management
------------------------------------------------------------
The pantry management view allows users to manage food items currently
available in the household pantry.

Users can:

- add pantry items
- edit pantry items
- remove pantry items
- view expiry dates
- view pantry waste-risk status

This view supports the pantry reuse element of the recommendation engine.


API Integration
------------------------------------------------------------
The frontend communicates with the backend using the following API
endpoints:

- /api/foods
- /api/pantry
- /api/recommendations
- /api/recipes/{id}

These endpoints are called using the browser fetch() API.

The backend base URL is currently configured as:

    const API_BASE = "http://localhost:8080";

This can be modified in the App.jsx file if the backend is hosted
elsewhere.
